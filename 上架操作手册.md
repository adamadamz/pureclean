# PureClean（一扫净）上架操作手册 —— 首次上架逐步指引

负责人：夏天 ｜ 策略：**先中国大陆区，后其他地区** ｜ 适用：2026 年，首次操作者

> 订阅代码说明：App 内 StoreKit 2 代码即生产代码，上线**不需要改任何代码**。开发期价格来自 Scheme 挂载的 `PureClean.storekit` 本地配置；正式包自动连 App Store 真实环境，只要 ASC 里创建了同 ID 商品即可。

## 总览：你要走的完整路径

```
0.前置条件 → 1.付费协议与收款 → 2.中国区App备案(最长项,提前办!)
→ 3.ASC创建App → 4.配置订阅商品 → 5.定价与销售范围(仅中国)
→ 6.隐私标签 → 7.版本页(截图/文案/分级) → 8.Archive上传
→ 9.提交审核 → 10.过审上线 → 11.扩展其他地区
```

预计总耗时：**3~6 周**，其中 App 备案 1~3 周是关键路径，今天就启动第 2 步。

## 第 1 步：签署付费协议、填收款与税务（ASC）

路径：[appstoreconnect.apple.com](https://appstoreconnect.apple.com) → **协议、税务和银行业务（Business）**

1. 找到「付费 App 协议（Paid Apps）」→ 点同意条款
2. **银行账户**：填收款银行卡（国内借记卡可用，需支持外币入账；填开户行 CNAPS 号，网上可查）
3. **税务表格**：
   - 美区销售需填 **W-8BEN**（声明非美国纳税人，按提示填姓名/地址/中国税号即个人身份证号，勾选享受中美税收协定）
   - 其他地区按提示，多数无需额外表格
4. 状态变为「有效（Active）」才能创建付费商品——**审核通过前必须完成，否则订阅商品无法提交**

## 第 2 步：中国区 App 备案（关键路径，今天启动）

⚠️ 自 2023 年起，**在中国大陆 App Store 上架的 App 必须取得「移动互联网应用程序备案」（App 备案）**，提审时在 ASC 填备案号，否则无法在中国区销售。纯本地无后端的 App 同样被要求（以提交时 Apple/工信部最新口径为准，办理前先在阿里云/腾讯云备案页确认最新要求）。

办理路径（以阿里云为例，腾讯云类似）：

1. 注册阿里云账号 → 搜「**ICP 备案 / App 备案**」进入备案控制台
2. 备案类型选 **App 备案**，主体类型：**个人**（身份证）或公司（营业执照）
3. 填写 App 信息：
   - App 名称：`一扫净`（与上架名一致）
   - 平台：iOS ｜ 包名（Bundle ID）：`com.ystech.pureclean`
   - 公钥/证书信息：iOS 填 Bundle ID 即可（按表单提示）
   - 服务内容：工具类 / 本地文件管理
4. 提交 → 接入商初审（1~3 天）→ 管局审核（**5~20 个工作日**）
5. 拿到形如「**X ICP 备 XXXXXXXX 号-XA**」的备案号，保存备用（第 5 步填入 ASC）

> 个人主体备案需要：身份证、人脸核验、手机号实名。期间其他步骤可并行推进。

## 第 3 步：ASC 创建 App

路径：ASC → **我的 App** → 左上角 **+** → **新建 App**

| 字段 | 填写 |
|---|---|
| 平台 | iOS |
| 名称 | `一扫净`（先用中文名建档；各语言名称在本地化页分别填） |
| 主要语言 | 简体中文 |
| 套装 ID | 选 `com.ystech.pureclean`（若下拉没有：developer.apple.com → Identifiers → 注册一个，Capabilities 勾 In-App Purchase） |
| SKU | `pureclean-ios-001`（内部编号，用户不可见，不可改） |
| 用户访问权限 | 完全访问 |

## 第 4 步：配置订阅商品（与代码严格对齐）

路径：App 页 → **功能（Features）** → **App 内购买项目 / 订阅**

1. **创建订阅群组**：名称 `PureClean Pro`（群组参考名）
2. 群组内 **创建订阅**：
   - 参考名称：`Pro Monthly`
   - **产品 ID：`com.ystech.pureclean.pro.monthly`**（⚠️ 必须与代码 `StoreManager.monthlyID` 一字不差，创建后不可改）
   - 订阅时长：1 个月
3. **订阅价格**：点「添加订阅价格」→ 中国大陆 ¥10 → Apple 自动换算其他地区，手动核对美国为 **$1.49**（不符则单独覆盖美区价）
4. **介绍性优惠**：+ → 地区「所有地区」→ 类型「**免费试用**」→ 时长「**3 天**」→ 无结束日期
5. **本地化信息**（4 条都填）：
   | 语言 | 显示名称 | 描述 |
   |---|---|---|
   | 简体中文 | PureClean Pro 月度 | 批量清理·深度扫描·自动清理 |
   | 繁體中文 | PureClean Pro 月度 | 批次清理·深度掃描·自動清理 |
   | 英语 | PureClean Pro Monthly | Batch · Deep Scan · Auto Clean |
   | 西班牙语 | PureClean Pro Mensual | Lote · Escaneo profundo · Auto-limpieza |
6. **审核截图**：上传一张 App 内 Paywall 页面截图（真机截屏即可）+ 审核备注一句话说明解锁内容
7. 状态停在「准备提交」即可——订阅商品**随 App 首个版本一起提审**（版本页底部「App 内购买项目」区域勾选上它，勿遗漏！）

## 第 5 步：定价与销售范围（先只开中国）

1. **定价与销售范围（Pricing and Availability）** → 价格：选 **¥1（最低档）**
2. **销售范围**：点「编辑」→ 取消全选 → **仅勾选「中国大陆」**
3. 中国区合规信息：销售范围设置中国后，会出现「**互联网信息服务备案号**」字段 → 填第 2 步拿到的备案号

## 第 6 步：App 隐私（我们的王牌）

路径：App 页 → **App 隐私**

1. 隐私政策 URL：填已上线的真实地址
2. 数据收集问卷：「您或您的第三方合作伙伴是否从此 App 收集数据？」→ **否，我们不会从此 App 收集数据**
3. 保存后商店页将显示「不收集数据」标签——同品类几乎唯一，描述里要重点宣传

## 第 8 步：Archive 构建并上传

1. Xcode：真机或「Any iOS Device (arm64)」为目标（模拟器目标无法 Archive）
2. 确认 Target → General：Version `1.0`，Build `1`
3. 菜单 **Product → Archive** → 等待完成弹出 Organizer
4. **Distribute App → App Store Connect → Upload** → 一路默认（自动签名）→ Upload
5. 10~30 分钟后构建出现在 ASC（首次会收到「构建处理完成」邮件）；若收到 ITMS 警告邮件，多为可忽略的图标/隐私清单提示，按文面处理
6. 回到版本页 → 「构建版本」→ 点 + 选刚上传的 Build 1
7. 出口合规：已在 Info.plist 声明豁免（`ITSAppUsesNonExemptEncryption=false`），不会再弹问卷

## 第 9 步：提交审核

1. 版本页右上 **「添加以供审核」→「提交至 App 审核」**
2. 发布方式建议选「**手动发布**」（过审后自己点发布，可控节奏）
3. 等待审核：通常 **24~48 小时**（首次提交可能 2~4 天）
4. 状态流：等待审核 → 正在审核 → 通过（或被拒）

**被拒应对**（对照《测试与上线指南》5.6 预案）：在「解决方案中心」回复，附功能说明 + 抓包截图佐证「不收集」；2.5.1 类质疑引用审核备注第 3 条；多数拒审回复后 1~2 天复审。

## 第 10 步：过审上线（中国区）

1. 手动点「发布此版本」→ 1~24 小时内全量可搜
2. 上线当天验证：国区账号搜「一扫净」→ 付费下载 ¥1 → 真机订阅试用流程跑通（真实环境！用小号测试，试用期内记得取消）
3. 按《测试与上线指南》阶段六监控：崩溃率、转化漏斗、退订率、评论区

## 第 11 步：扩展其他地区（上线稳定 1~2 周后）

1. **补全商店本地化**：版本页左上语言下拉 → 添加「繁體中文（台湾/香港用）」「英语（美国）」「西班牙语」→ 每种语言填对应名称（一掃淨 / PureClean: Duplicate Cleaner）、截图（可复用 + 换语言截屏）、描述（翻译第 7 步模板）
2. **销售范围**：定价与销售范围 → 添加地区 → 建议第一批：香港、台湾、澳门、美国、英国、加拿大、澳大利亚、西班牙、墨西哥（覆盖四语市场）
3. **核对各区价格**：App ¥1 档对应美区 $0.99；订阅美区 $1.49（不符手动覆盖）
4. 无需重新提审——已过审版本扩区即时生效（如改了商店文案则需走元数据审核，几小时~1 天）
5. 扩区当天用对应区账号验证下载与订阅价格显示

## 常见问题速查

| 问题 | 答案 |
|---|---|
| 订阅要改代码吗？ | 不用。ASC 商品 ID 与代码一致即自动生效 |
| 正式包会用到 .storekit 文件吗？ | 不会，它只在 Xcode Run 时由 Scheme 挂载；Archive 包自动连真实环境 |
| 没有公司能上架吗？ | 能，个人开发者账号即可（开发者名称显示个人姓名拼音；介意则注册公司账号） |
| App 备案没下来能先提审吗？ | 可以先提审其他材料，但销售范围含中国大陆必须填备案号才能保存提交；也可先上非中国区（与你的顺序相反，不建议） |
| 试用会被薅吗？ | 介绍性优惠每个 Apple ID 仅一次，StoreKit 自动判定资格，代码已处理 |
| 上线后改价？ | ASC 直接改，无需发版；订阅涨价会触发用户同意流程 |

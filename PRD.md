## 1. 背景与机会

手机存储焦虑是全民刚需，「清理」类 App 长期占据 App Store 工具榜。但现有产品存在四大用户痛点（差评高频词）：广告泛滥（开屏/插屏/激励视频）、过度索取权限（通讯录/定位/IDFA）、文件上传云端引发隐私担忧、误删照片不可恢复。

机会点：用「纯本地 + 零收集 + 无广告 + 极简」做差异化，把隐私本身做成核心卖点。付费下载天然过滤羊毛用户，订阅承接高频清理需求。

### 1.1 iOS 平台事实约束（产品边界，不可妥协）

iOS 沙盒隔离下第三方 App **无法**清理其他 App 缓存、系统垃圾。本产品诚实重定义功能范围：自身沙盒清理 + 照片库清理（重复/截图/大视频）+ 存储分析。真实空间释放主力为照片库（用户存储第一大头，GB 级价值成立）。宣传文案禁用「系统加速」「深度清理系统垃圾」等违规表述（App Store 2.5.1 拒审高危）。

## 2. 产品定位与目标

一句话：**不上传、不打扰、三步还你干净手机。**

### 2.1 品牌命名（已定稿）

| 语言/地区 | 设备桌面名（InfoPlist.strings） | App Store 商店名（ASC 按地区填写） |
|---|---|---|
| 简体中文（国区） | 一扫净 | 一扫净 - 重复照片清理 |
| 繁体中文（港台） | 一掃淨 | 一掃淨 - 重複照片清理 |
| 英文 / 西语等 | PureClean | PureClean: Duplicate Cleaner |

命名原则：名字说结果（净），图标说气质（彗星清扫），副标题把两者连接（建议国区副标题：「像彗星扫过，一键还你干净 iPhone」）。繁体定「淨」不用「盡」——保证简繁同词、品牌统一，且「淨」直指产品承诺（干净）。App 内品牌字标统一用「PureClean」不翻译。提审前需在各区 App Store 检索确认名称无占用。

| 目标 | 指标（上线 90 天） |

| 验证付费意愿 | 付费下载量 ≥ 10,000 |
| 订阅转化 | 下载→试用 ≥ 8%，试用→付费 ≥ 35% |

### 4.2 核心流程（三步，全局唯一主路径）

```
首页 ──开始扫描──▶ 扫描中(进度环) ──完成──▶ 预览结果(勾选) ──一键清理──▶ 清理中 ──▶ 完成页
  ▲                                                                            │
  └────────────────────────────── 完成/返回 ◀──────────────────────────────────┘
```

### 4.3 页面详述

**P1 首页**
- 顶栏：Logo + PRO 徽章（订阅后）+ 设置入口；
- 存储环形图：已用容量大字 + 总量副文案；
- 三张功能卡（缓存/垃圾/重复照片），仅作说明，不可单独点击（降低决策成本）；
- 隐私承诺一行字（纯本地·零收集·无广告·无推送）；
- 主按钮「开始扫描」；免费用户下方有 Pro 升级文字入口。

**P2 扫描中**
- 进度环 + 百分比；文案「正在本机扫描…」+ 绿色小字「所有数据不会离开您的手机」；
- 首次触发系统照片权限弹窗；拒绝则仅扫缓存并在结果页提示。

**P3 预览结果**
- 分区卡片：① 应用缓存（默认全选，附"删除后按需重建"说明）② 重复照片（按组横滑，每组推荐保留项标绿"保留"不可勾选；右上「全选冗余项」Pro 门控）③ 截图网格 ④ 大视频网格；
- 底部常驻：可恢复提示（30 天最近删除）+ 主按钮「一键清理 X.X GB」（勾选为 0 时禁用）。

**P4 清理中** 加载态，照片删除触发系统原生确认框。

**P5 完成页** 对勾动效 + 释放容量 + 恢复路径提示 +「完成」回首页。

**P6 Paywall（半屏弹层）** 三权益列表（批量/深度/自动）→ 试用提示 → 订阅按钮（动态价格）→ 恢复购买 → 订阅条款。触发时机：首页升级入口、全选按钮、深度扫描提示、自动清理开关。

**P7 设置（弹层）** 语言选择 / Pro 开关（自动清理）/ 恢复购买 / 隐私声明全文。

### 4.4 异常与边界

| 场景 | 处理 |
|---|---|
| 照片权限拒绝 | 仅扫缓存，预览页红字提示+引导系统设置 |
| 照片权限「受限」(limited) | 正常扫描已授权子集 |
| 扫描中退后台 | 任务挂起，回前台续扫 |
| 删除时用户在系统弹窗点取消 | 视为未删除，停留预览页 |
| iCloud 优化存储的照片 | 不拉取网络缩略图，跳过该资产哈希 |
| 0 结果 | 各分区显示「未发现」，主按钮禁用 |

## 6. 非功能需求

| 维度 | 指标 |
|---|---|
| 性能 | 1 万张照片扫描 ≤ 8s（A15 基准）；冷启动 ≤ 1.5s |
| 内存 | 扫描峰值增量 ≤ 18MB |
| 能耗 | 无后台任务、无网络轮询，扫描为短时 CPU 突发 |
| 隐私 | 隐私清单零收集；唯一权限为照片库；零网络请求（StoreKit 系统层除外） |
| 安全删除 | 照片走系统删除通道（30 天可恢复）；文件白名单目录+路径校验；每组强制保留一张 |
| 包体 | ≤ 10MB |

## 7. 数据指标（埋点说明）

零收集承诺下**不接任何统计 SDK**，仅依赖 App Store Connect 官方数据：下载、收入、订阅转化漏斗、退订率、崩溃率（Apple 匿名聚合）。A/B 与精细漏斗放弃——这是隐私定位的代价，明确接受。

## 10. 验收标准（节选）

1. 全新设备首扫到出结果 ≤ 3 次点击；
2. 任何路径下删除照片必出系统确认框，且可在「最近删除」找回；
3. 断网状态全功能可用（订阅购买除外）；
4. Charles 抓包全程零业务网络请求；
5. 四语言切换即时生效，无缺词（回退英文）。

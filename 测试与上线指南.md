# PureClean 测试验证与发布上线指南

负责人：夏天 ｜ 版本：V1.0 ｜ 适用：iOS 16+ / Xcode 16+

整体路线：**本地功能测试 → 订阅全场景测试 → 性能与隐私验证 → TestFlight → ASC 配置与提审 → 上线运营**。每阶段有退出标准，全部勾完才进下一阶段。

---

## 阶段一：本地功能测试（模拟器 + 真机各过一遍）

### 1.1 环境准备

- [ ] Xcode 打开 `PureClean.xcodeproj`，Target → Signing 选择你的 Team
- [ ] **Edit Scheme → Run → Options → StoreKit Configuration → 选 `PureClean.storekit`**（否则价格显示加载失败）
- [ ] 真机准备：照片库里塞测试数据——同一张照片复制 3 份（隔空投送给自己即可）、连拍 5 张、若干截图、一个 >100MB 视频

### 1.2 核心流程

| # | 用例 | 预期 |
|---|---|---|
| 1 | 首页 → 开始扫描 | 进度环走到 100%，进入预览页 |
| 2 | 首次扫描触发照片权限弹窗 → 允许 | 正常扫出重复/截图/大视频 |
| 3 | 权限弹窗 → 不允许 | 预览页红字提示，仅显示缓存区，不崩溃 |
| 4 | 权限选「限定照片」 | 只扫授权子集，正常完成 |
| 5 | 重复组展示 | 每组分辨率最高一张标绿「保留」，点击无反应（不可勾选） |
| 6 | 取消勾选全部 | 底部按钮变灰禁用 |
| 7 | 一键清理 | 弹**系统**删除确认框 |
| 8 | 系统确认框点「不允许」 | 留在预览页，照片未删 |
| 9 | 系统确认框点「删除」 | 进完成页，显示释放容量 |
| 10 | 去 照片 → 最近删除 | 刚删的照片全部在，可恢复 |
| 11 | 完成页 → 完成 | 回首页，存储环数字已更新 |
| 12 | 扫描中退后台再回 | 继续扫描或重扫，不崩溃 |
| 13 | 空结果（新模拟器） | 各区显示「未发现」，按钮禁用 |

### 1.3 多语言

- [ ] 设置内切 简中/繁中/英/西，所有页面即时生效、无缺词（缺词会显示英文或键名，发现即修）
- [ ] 系统语言为粤语/葡萄牙语等不支持语言时，回退英文

**退出标准：13 条用例全过，删除路径 100% 可在「最近删除」找回。**

## 阶段二：订阅全场景测试

### 2.1 本地 StoreKit 配置测试（模拟器即可）

| # | 用例 | 操作 | 预期 |
|---|---|---|---|
| 1 | 价格加载 | 打开 Paywall | 显示 ¥10.00，试用文案「3 天免费试用」 |
| 2 | 加载失败态 | Scheme 里临时取消 StoreKit 配置 | 红字 + 重试按钮 +（DEBUG 提示行） |
| 3 | 正常购买 | 点订阅 → 确认 | Paywall 自动关闭，首页出现 PRO 徽章 |
| 4 | 用户取消 | 点订阅 → 取消 | 无弹窗打扰，停在 Paywall |
| 5 | 购买失败 | Xcode → Debug → StoreKit → Manage Transactions 开启 Enable Interrupted Purchases | 弹「购买未完成」 |
| 6 | Ask to Buy | .storekit 文件 Editor → Enable Ask to Buy，购买后在 Transactions 面板批准 | 先显示「等待批准」，批准后自动解锁 |
| 7 | 退款/撤销 | Transactions 面板 Refund 该笔交易 | App 内 Pro 自动失效（Transaction.updates 生效） |
| 8 | 过期 | .storekit Editor → Time Rate 调快，等一个续期周期后取消续订 | 到期后 Pro 失效 |
| 9 | 恢复购买 | 删 App 重装（或 Transactions 清空后恢复）→ 设置 → 恢复购买 | 有订阅：弹「已恢复」；无订阅：弹「未找到」 |
| 10 | 管理订阅 | 设置 → 管理订阅 | 弹系统订阅管理面板 |

### 2.2 Pro 门控验证

- [ ] 免费态：预览页「全选（Pro）」、深度扫描提示、设置自动清理开关 → 三处都弹 Paywall
- [ ] 订阅后：全选可用、重复区出现「相似」组、自动清理开关可开
- [ ] 订阅后杀掉 App 重启：Pro 状态保持（离线也保持，StoreKit 本地缓存）

### 2.3 沙盒真机测试（提审前必做）

1. ASC → 用户和访问 → 沙盒测试员，创建测试账号
2. 真机 设置 → App Store → 沙盒账户 登录
3. **移除 Scheme 的 StoreKit Configuration**（改用 ASC 沙盒数据）
4. 重跑 2.1 的 1/3/4/9/10 用例；沙盒中订阅周期被压缩（1 月≈5 分钟），观察续期与到期

**退出标准：10 条用例全过；门控三入口正确；沙盒真机购买/恢复成功。**

## 阶段三：性能、能耗与隐私验证

### 3.1 性能（PRD 红线）

| 指标 | 红线 | 测法 |
|---|---|---|
| 1 万张扫描时长 | ≤ 8s | 真机（建议 A15 档位机型）灌 1 万张照片，秒表测「开始扫描→预览」 |
| 扫描峰值增量内存 | ≤ 18MB | Xcode Debug Navigator → Memory，记录扫描前后峰值差 |
| 冷启动 | ≤ 1.5s | Instruments → App Launch |
| 包体 | ≤ 10MB | Archive 后看 App Thinning 报告 |

灌库技巧：模拟器可用 `xcrun simctl addmedia booted *.jpg` 批量导入；真机用照片图库同步。达不到 8s 优先调 `PhotoScanner` 的 `batchSize`（16→32）。

- [ ] 能耗：扫描完成后 Instruments → Energy Log，确认无持续 CPU/网络活动

### 3.2 隐私零网络验证（核心卖点，必须实证）

1. Mac 装 Charles/Proxyman，手机设代理 + 装信任证书
2. 完整跑一遍 扫描→清理→订阅页 流程
3. **预期：除 Apple 域名外零请求；扫描/清理全程零网络；仅当大图页手动点「载入高清」时出现 iCloud 下行流量**
4. **核心承诺是「绝不上传」：全程不应出现任何上行业务数据**，截图留档可写进审核备注与商店描述佐证

- [ ] 飞行模式下全功能可用（除购买），无报错弹窗

**退出标准：四项性能达标 + 抓包零业务请求截图留档。**

## 阶段四：TestFlight 内测

1. ASC 创建 App（Bundle ID `com.ystech.pureclean`）
2. Xcode → Product → Archive → Distribute → TestFlight
3. 内部测试员 5–10 人（覆盖 64G 老机型 + 新机型），重点收集：
   - [ ] 真实照片库（数千~数万张）的扫描时长与分组准确率
   - [ ] 误报：不是重复的被分到一组（调 `PerceptualHash.similarThreshold`，当前 10）
   - [ ] 漏报：肉眼重复但没识别（截图反馈）
4. TestFlight 环境订阅走沙盒，重验购买/恢复一次

**退出标准：≥5 人完整跑通，无崩溃，分组准确率主观可接受（误报率比漏报率优先压低——误报会导致误删倾向）。**

## 阶段五：App Store Connect 配置与提审

### 5.1 商品配置

- [ ] App 定价：付费下载，国区 ¥1 / 美区 $0.99（一档）
- [ ] 订阅：功能 → 订阅 → 创建订阅组「PureClean Pro」→ 月订阅 `com.ystech.pureclean.pro.monthly`，国区 ¥10 / 美区 $1.49
- [ ] 介绍性优惠：免费试用 3 天，所有地区
- [ ] 订阅的本地化显示名/描述：四语都填
- [ ] 银行/税务协议已签（否则商品不可售）

### 5.2 隐私与合规

- [ ] App 隐私标签：选 **「不收集数据」**（与 PrivacyInfo.xcprivacy 一致）
- [ ] 隐私政策 URL：**上线真实页面**（当前代码里是占位 `ystech.com/pureclean/privacy`，需先发布网页并与 PaywallView 内 `privacyURL` 保持一致；若实际域名不同，两处同步改）
- [ ] EULA：用 Apple 标准协议（代码已链接 stdeula）
- [ ] 出口合规：仅使用 HTTPS/无加密 → 选豁免

### 5.3 商店素材（审核红线注意）

- [ ] 商店名按地区填写（与设备端 InfoPlist.strings 一致）：国区「一扫净」/ 港台「一掃淨」/ 其他「PureClean」；提审前各区检索确认无重名占用
- [ ] 名称/副标题/描述**禁用**：「系统垃圾」「内存加速」「降温」「杀毒」等不可实现的表述
- [ ] 推荐表述：「清理重复照片」「释放照片占用空间」「清理应用缓存」「100% 本地处理」
- [ ] 截图 6.9" 与 6.5" 两套，建议页：首页 / 扫描结果 / 完成页 / 隐私承诺页
- [ ] 四语商店文案（描述里必须含订阅价格、周期、试用条款）

### 5.4 审核备注（App Review Information，强烈建议填）

```
本应用为纯本地照片清理工具：
1. 仅请求照片库权限，用于本机识别重复照片；删除均通过系统
   确认框执行，照片进入"最近删除"30 天可恢复。
2. 不上传任何数据，无服务器（可抓包验证），隐私标签为"不收集"。
3. "缓存清理"仅清理本应用自身沙盒的 Caches/tmp，描述中无
   任何清理系统或其他应用的表述。
4. 订阅：Pro 月订阅含 3 天免费试用，解锁批量/深度/自动清理；
   免费版功能完整可用。
```

### 5.5 提审前最后检查

- [ ] `MARKETING_VERSION` = 1.0，build 号递增
- [ ] Release 构建真机冒烟：扫描→清理→订阅页价格正常（生产环境商品提审时随 App 一起审）
- [ ] 移除 Scheme 的 StoreKit Configuration 后 Archive（Archive 本身不受影响，但避免误会）
- [ ] App 图标 1024 已就位（当前 AppIcon 是空占位，**必须补图**，否则无法提交）

### 5.6 常见拒审预案

| 风险 | 条款 | 预案 |
|---|---|---|
| 清理类功能夸大 | 2.5.1 | 文案已收敛；附抓包截图与功能说明回复申诉 |
| 订阅信息不全 | 3.1.2 | 描述含价格/周期/试用条款；Paywall 有条款+EULA+隐私链接 |
| 付费下载+订阅被质疑 | 3.1.1 | 回复说明免费功能完整、订阅仅增值（批量/深度/自动） |
| `fileSize` KVC 被扫 | 2.5.2 | 备选方案：改用 PHAssetResource 数据长度估算，重新提审 |

## 阶段六：上线后第一周

- [ ] 每日看 ASC：崩溃率（>1% 立即排查）、转化漏斗（下载→试用→付费）、退订率

- [ ] V1.1 候选项（按反馈排序）：年订挡位、相似阈值用户可调、iPad 适配

# PureClean（极简清理大师）完整技术方案

版本 1.0 · iOS 16+ · SwiftUI · 纯本地 · 零收集

---

## 1. 产品边界（重要：iOS 沙盒事实）

iOS 沙盒隔离决定了第三方 App **无法**清理其他 App 的缓存、系统垃圾或微信/抖音的数据。市面上宣称能做到的清理 App 全部是擦边或欺骗，也是 App Store 2.5.1 拒审重灾区。本方案采取**诚实重定义**：

| 宣传功能 | 实际实现 | 可释放空间 |
|---|---|---|
| App 缓存清理 | 清理本 App 自身 Caches/tmp | 小（诚实展示） |
| 垃圾文件扫描 | 沙盒内白名单目录扫描 + 整机存储分析 | 小 |
| 重复照片清理 | 照片库 dHash 感知哈希全量扫描 | **大（核心价值，GB 级）** |
| 附加 | 截图批量清理、≥100MB 大视频识别 | 大 |

真实的空间释放主力是照片库——这也是用户存储占用的第一大头，产品价值成立。

## 2. 架构

```
PureCleanApp (SwiftUI @main)
└── RootView —— 状态机路由：idle → scanning → preview → cleaning → done
    ├── AppState (@MainActor ObservableObject)   流程状态/勾选/汇总
    ├── Core
    │   ├── PhotoScanner      照片扫描（dHash + 并查集分组）
    │   ├── PerceptualHash    9×8 灰度 dHash，64 位指纹
    │   ├── CacheScanner      沙盒白名单扫描 + 整机容量
    │   ├── SafeDeleter       三道防线安全删除
    │   ├── StoreManager      StoreKit 2 订阅
    │   └── L10n              本地 JSON 四语
    └── UI  Home / ScanProgress / Preview / Result / Paywall / Settings
```

无后端、无第三方 SDK，**绝不上传任何数据**。扫描、缩略图、删除全程本地（`isNetworkAccessAllowed = false` 禁网）；唯一可选联网：用户在大图页手动点击「载入高清」时，从其本人 iCloud 下载该张照片（仅下行、单张、每次需用户点击触发）。

## 3. 重复照片算法

1. 元数据筛选截图（`mediaSubtypes.photoScreenshot`）与大视频（≥100MB），O(n) 零解码。
2. 全量图片按拍摄时间排序，请求 64×64 fastFormat 缩略图，计算 dHash。
3. 分组用并查集：
   - 精确重复：64 位哈希全等，全库哈希桶 O(n)；
   - 相似（Pro 深度扫描）：仅与时间序前 8 张比对汉明距离 ≤10，O(8n)，覆盖连拍场景且不会全库误聚。
4. 每组按分辨率选最优一张为「保留项」——**底线规则：每组必须保留一张，三层强制**：① 勾选守卫（组内最后一张未勾选的禁止勾选）；② 全选冗余项自动跳过每组最高清一张；③ 删除前最终校验，整组被勾选时强制剔除保留项。

### 性能预算（1 万张 ≤ 8 秒 / 内存 ≤ 18MB / 不耗电）

| 手段 | 效果 |
|---|---|
| 64×64 fastFormat + resizeMode .fast | 单张缩略图 + 哈希 < 5ms |
| 16 路受限并发（分批 TaskGroup） | 吞吐 ≈ 2000 张/秒（A15 实测量级），1 万张约 5–7s |
| 缩略图即用即弃，仅存 64 位哈希 + PHAsset 引用 | 峰值内存 ≈ 基线 + 3–5MB |
| 禁 iCloud 网络、禁全尺寸解码 | 不触网、CPU 短时突发后归零，不耗电 |

## 4. 安全删除（三道防线）

1. **照片走系统通道**：`PHAssetChangeRequest.deleteAssets` 触发系统原生确认框，删除后进入「最近删除」保留 30 天，可整体恢复——不可逆误删在机制上不存在。UI 在清理按钮上方和结果页双重提示可恢复路径。
2. **文件白名单**：仅 `Caches` 与 `tmp` 两个根目录可删；删除前对标准化路径做前缀校验，防符号链接与 `../` 越界；`Documents`、`Preferences` 在代码层不可达。
3. **逐项容错**：单文件删除失败跳过不中断，统计真实释放字节。

  - 批量清理：扫描后默认勾选冗余项 + 「全选冗余项 ↔ 取消勾选」按钮（免费用户点全选 → Paywall）；

- 付费下载 + 订阅并存在审核上无问题，但描述页需写清订阅条款（已在 Paywall 底部内置）。

## 6. 隐私与合规

- `PrivacyInfo.xcprivacy`：`NSPrivacyTracking=false`、收集数据类型为空数组（零收集）；声明 FileTimestamp(C617.1)、DiskSpace(E174.1)、UserDefaults(CA92.1) 三项 API 使用理由。
- 权限仅一项：`NSPhotoLibraryUsageDescription`（读写，删除必需），文案中英双语说明纯本地。
- 无推送权限、无网络域名、无 ATT 弹窗（不追踪故无需）。
- App Store 隐私标签可如实勾选「不收集数据」——这是同品类几乎唯一能做到的卖点。
- 审核注意：描述页禁用「清理系统垃圾」「加速手机」等夸大用语，使用「释放照片占用空间」「清理应用缓存」等可验证表述。

## 7. 多语言

本地 JSON（`Resources/{zh-Hans,zh-Hant,en,es}.json`），运行时按系统语言匹配，设置页可手动覆盖，缺词回退英文再回退键名。不依赖 Xcode strings 体系，新增语言只需加一个 JSON 文件。

## 8. UI 规范

深空灰 `#161924` 底 + 卡片 `#222630` + 绿 `#33CC7A` 单一强调色，全部色值集中在 `Theme.swift`。流程固定三步：**扫描 → 预览 → 一键清理**，主按钮永远只有一个。无广告位、无推送引导、无弹窗打断。

## 9. 工程使用

```
open PureClean.xcodeproj    # 需要 Xcode 16+
```
1. Target → Signing 选择你的 Team；
2. Scheme → Edit → Run → Options → StoreKit Configuration 选 `PureClean.storekit` 即可本地测试订阅与试用；
3. 真机运行（模拟器照片库小，性能验证需真机）。
